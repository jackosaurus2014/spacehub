TIER 1 — CRITICAL SECURITY FIXES (do immediately)
=============================================================

1.  FIX DEAL ROOM AUTHENTICATION (Critical Security)
    Every deal-rooms API route authenticates via self-reported email parameter
    instead of getServerSession(). Any user can impersonate any other user by
    passing their email. Affects 6 route files:
      - src/app/api/deal-rooms/route.ts
      - src/app/api/deal-rooms/[id]/route.ts
      - src/app/api/deal-rooms/join/route.ts
      - src/app/api/deal-rooms/[id]/documents/route.ts
      - src/app/api/deal-rooms/[id]/members/route.ts
      - src/app/api/deal-rooms/[id]/nda/route.ts
    FIX: Replace all email params with getServerSession(authOptions), use session.user.id.
STATUS: DONE (commit 2dfafc5) — All 6 routes converted to session-based auth.
    Added createdByUserId + userId fields to DealRoom/DealRoomMember Prisma models.
    Deal rooms page uses useSession() instead of email input form.
    Added Zod schemas: dealRoomCreateSchema, dealRoomJoinSchema, dealRoomUpdateSchema,
    dealRoomInviteSchema, dealRoomDocumentSchema.

2.  FIX INVESTMENT THESIS AUTHENTICATION (Critical Security)
    POST /api/investment-thesis has NO authentication check. Any anonymous user
    can query full company financial data AND trigger expensive Anthropic API calls.
    FIX: Add getServerSession() check + tier-based access control + specific rate limit.
STATUS: DONE (commit 2dfafc5) — Added getServerSession() auth check + Zod validation
    via investmentThesisSchema. Rate limited to 5 requests/hr in middleware.

3.  FIX XSS IN BLOG CONTENT RENDERING (High Security)
    src/app/blog/[slug]/page.tsx line 166 uses dangerouslySetInnerHTML={{ __html: post.content }}
    with RSS-sourced content that may not be fully sanitized.
    FIX: Sanitize post.content with DOMPurify or sanitize-html before rendering.
STATUS: DONE (commit 2dfafc5) — Added sanitize-html with strict SAFE_HTML_CONFIG.
    Sanitizes all RSS content before rendering via dangerouslySetInnerHTML.

4.  FIX XSS IN AI INSIGHTS MARKDOWN RENDERER (High Security)
    src/app/ai-insights/[slug]/page.tsx has custom markdownToHtml() that generates
    <a> tags from content without sanitizing href attributes (javascript: URLs possible).
    Rendered via dangerouslySetInnerHTML.
    FIX: Use react-markdown + remark-gfm (already installed) instead of custom converter,
    or sanitize output with DOMPurify.
STATUS: DONE (commit 2dfafc5) — Deleted entire custom markdownToHtml() function (~120 lines).
    Replaced with <ReactMarkdown remarkPlugins={[remarkGfm]}> with Tailwind-styled
    component overrides. No more dangerouslySetInnerHTML.

5.  ADD ZOD VALIDATION TO 14 API ROUTES (High Security)
    These routes accept user input via request.json() but use manual checks instead
    of Zod schemas. Inconsistent validation increases risk of logic bypasses:
      - /api/subscription (tier, interval)
      - /api/reports/generate (reportType, config)
      - /api/investment-thesis (companySlug)
      - /api/deal-rooms/* (6 routes — name, email, accessCode, etc.)
      - /api/marketplace/copilot (message, history)
      - /api/marketplace/match (rfqId)
      - /api/launch-day/*/polls, */reactions, */polls/vote
      - /api/marketplace/verify/admin
    FIX: Create Zod schemas in validations.ts and use validateBody() pattern.
STATUS: DONE (commit 2dfafc5) — Added 8+ new schemas to validations.ts:
    investmentThesisSchema, subscriptionActionSchema, reportGenerateSchema,
    marketplaceCopilotSchema, marketplaceMatchSchema, marketplaceVerifyAdminSchema,
    launchPollSchema, launchPollVoteSchema. Deal room schemas in task 1.

6.  FIX CSRF BYPASS FOR BEARER HEADERS (Medium Security)
    src/middleware.ts lines 216-219: ANY request with "Authorization: Bearer <anything>"
    bypasses CSRF, even if the token is fake/invalid.
    FIX: Only bypass CSRF for paths that actually validate Bearer tokens (/api/v1/*, cron endpoints).
STATUS: DONE (commit 2dfafc5) — CSRF bypass now validates: cronSecret && token === cronSecret
    plus path whitelist check for /api/v1/* and cron endpoints only.

7.  FIX CRON SCHEDULER HARDCODED LOCALHOST (High — Breaks Production)
    src/lib/cron-scheduler.ts line 4:
      const BASE_URL = `http://localhost:${process.env.PORT || 3000}`;
    This breaks all cron jobs in Railway production environment.
    FIX: Use process.env.NEXT_PUBLIC_APP_URL || `http://localhost:${process.env.PORT || 3000}`
STATUS: DONE (commit 2dfafc5) — Changed to use NEXT_PUBLIC_APP_URL with localhost fallback.
